# План: Панель инструментов VPS в блоке "Пульс"

## 1. Текущее состояние

Блок "СЕРВЕР VPS" сейчас показывает:
- Статус (активен/нет) + heartbeat
- CPU/RAM/Диск
- База данных (размер, свечи, стаканы, ликвидации)
- Пары VPS с trust_score
- ⚙️ настройки (host/port/apiKey)

**Чего не хватает:**
- Нет кнопок действий (валидация, тесты)
- Нет интеграции с эндпоинтами VPS (`/data/validate`, `/data/gaps`, `/telegram/test`)
- Ликвидации = 0 (непонятно: нет данных или баг)
- Нет возможности "пощупать" данные — проверить их корректность

### 📡 3.3 Сверка ТФ (кросс-таймфрейм) — НОВАЯ ЛОГИКА

**Суть:** самый мощный инструмент верификации. Проверяет, что данные на разных таймфреймах согласованы.

**Алгоритм:**
1. Запросить 1m свечи за диапазон (напр., последние 500 минут) через `/api/candles`
2. Агрегировать их в 5m свечи:
   - `open` = open первой 1m свечи в окне
   - `high` = max(high) всех 1m свечей в окне
   - `low` = min(low) всех 1m свечей в окне
   - `close` = close последней 1m свечи в окне
   - `volume` = sum(volume) всех 1m свечей в окне
3. Запросить существующие 5m свечи за тот же диапазон через `/api/candles`
4. Сравнить покадрово — найти расхождения
5. Показать результат

**UI:**
- Выпадающий список пар
- Выбор "базового ТФ" (1m) и "целевого ТФ" (5m/15m/1h)
- Кнопка "📡 Сверять"
- Результат:
  ```
  BTC/USDT: 1m → 5m
  ┌─────────┬──────────┬──────────┬──────────┐
  │ Время   │ Совпадает │ Расхожд. │ Ошибка % │
  ├─────────┼──────────┼──────────┼──────────┤
  │ 15:00   │ ✅       │ 0.01%    │ 0.01%    │
  │ 15:05   │ ✅       │ 0.00%    │ 0.00%    │
  │ 15:10   │ ❌       │ 1.23%    │ 1.23%    │ ← проблема!
  └─────────┴──────────┴──────────┴──────────┘
  Общий рейтинг: 99.8%
  ```

**Почему это ценно:**
- Обнаруживает "битые" свечи на старших ТФ
- Проверяет корректность агрегации на VPS
- Даёт численную метрику качества данных

**Где выполняется:** на фронтенде (React). VPS отдаёт только сырые свечи, агрегация и сравнение — на клиенте.

## 6. Альтернативы и компромиссы

### Вариант A: Все кнопки в одном блоке (рекомендую)
**Плюсы:** всё в одном месте, удобно
**Минусы:** блок может стать большим

### Вариант B: Кнопки в модальном окне
По клику на ⚙️ открывается модалка с настройками + инструментами
**Плюсы:** не занимает место
**Минусы:** нужно открывать модалку каждый раз

### Вариант C: Кнопки только для самых частых действий
Оставить только "Валидация" и "Сверка ТФ", остальное в подменю
**Плюсы:** компактно
**Минусы:** меньше функциональности на виду
