## 1. `candles` — Свечи (OHLCV)

**Статус:** ❌ Повреждена  
**Приоритет:** 🔴 Критическая

Основная таблица для хранения исторических ценовых данных. Содержит open/high/low/close/volume для всех символов, таймфреймов (1m, 5m, 15m, 1h, 1d, 1W) и типов рынка (spot/futures). Сердце системы — на этих данных работают все индикаторы, сигналы и бэктесты.

**Поля:**
- `symbol` — торговый символ (BTC/USDT)
- `timeframe` — таймфрейм (1m, 5m, ...)
- `open_time` — время открытия свечи (Unix ms)
- `market_type` — spot / futures
- `open, high, low, close, volume` — OHLCV
- `is_closed` — флаг закрытия свечи
- `source` — источник данных (exchange, aggregated, futures_v3)
- `data_trust_score` — оценка качества данных (0-100)

**Индексы:** `uq_candles(symbol, timeframe, open_time)` — уникальность, `idx_candles_lookup(symbol, timeframe, market_type, open_time)` — быстрый поиск.

**Кто пишет:** `BingXRestClient.fetch_klines()` (backfill), `BingXFuturesWebSocket._on_kline()` (online), `BingXWebSocket._on_kline()` (online spot), `tf_aggregator` (агрегация).

**Кто читает:** `CandlesRepository` (все методы), `Backtester`, `SignalEngine`, `TAEngine`, `ChartView`.

## 2. `orderbook_snapshots` — Снапшоты стакана

**Статус:** ❌ Повреждена  
**Приоритет:** 🟡 Важная

Хранит снимки order book (bid/ask уровни, спред, дисбаланс) в реальном времени. Используется для анализа ликвидности и рыночного давления.

**Поля:**
- `symbol` — торговый символ
- `timestamp` — время снапшота (Unix ms)
- `market_type` — spot / futures
- `bids` — JSON массива bid-уровней
- `asks` — JSON массива ask-уровней
- `bid_volume` — суммарный объём bid
- `ask_volume` — суммарный объём ask
- `spread` — спред (ask - bid) / mid_price
- `mid_price` — средняя цена (bid + ask) / 2
- `imbalance` — дисбаланс (bid_volume - ask_volume) / (bid_volume + ask_volume)

**Кто пишет:** `OBProcessor` (из WebSocket depth20).

**Кто читает:** `MarketMetrics`, аналитические модули.

## 3. `trade_raw` — Сырые сделки

**Статус:** ✅ Цела  
**Приоритет:** 🟡 Важная

Каждая совершённая сделка на бирже: цена, объём, направление (buy/sell). Нужен для расчёта CVD (Cumulative Volume Delta) и анализа потока ордеров.

**Поля:**
- `symbol`, `trade_id`, `price`, `quantity`, `side` (buy/sell), `timestamp`

**Кто пишет:** `BingXWebSocket._on_trade()` / `BingXFuturesWebSocket._on_trade()`.

**Кто читает:** `VolumeEngine` (CVD), аналитические модули.

---

## 4. `market_metrics` — Рыночные метрики

**Статус:** ✅ Цела  
**Приоритет:** 🟡 Важная

Спотовые показатели: волатильность, объём, изменения цены за разные периоды. Агрегированные метрики для анализа состояния рынка.

**Кто пишет:** `DataVerifier`, периодические расчёты.

---

## 5. `futures_metrics` — Фьючерсные метрики

**Статус:** ✅ Цела  
**Приоритет:** 🟡 Важная

OI (Open Interest), Funding Rate, Premium Index, Basis. Отслеживает настроения и стоимость удержания позиций на фьючерсном рынке.

**Кто пишет:** `BingXRestClient.fetch_open_interest()`, `fetch_funding_rate()`, `BasisCalculator`.

## 6. `liquidations` — Ликвидации

**Статус:** ✅ Цела  
**Приоритет:** 🟡 Важная

События принудительного закрытия позиций (long/short, объём, цена). Ключевой индикатор cascading-эффектов и экстремальных движений.

**Поля:**
- `symbol`, `side` (long/short), `price`, `quantity`, `timestamp`, `total_value`

**Кто пишет:** `BingXRestClient.fetch_force_orders()` (polling REST).
