# Исследование глубины исторических данных BingX API

**Дата:** 2026-05-01  
**Инструмент:** [`scripts/test_bingx_depth.py`](scripts/test_bingx_depth.py)  
**Метод:** Бинарный поиск + точечная проверка по ключевым датам

---

## Результаты

### 1. v3 Swap Klines (`/openApi/swap/v3/quote/klines`)

| Таймфрейм | Самая ранняя свеча | Глубина (дней) |
|-----------|-------------------|----------------|
| **1m**    | 2025-03-03 03:57 UTC | ~424 дня |
| **5m**    | 2026-03-17 08:10 UTC | ~45 дней |
| **15m**   | 2026-04-17 01:15 UTC | ~14 дней |
| **1h**    | 2026-03-19 07:00 UTC | ~43 дня |
| **1d**    | 2025-05-21 00:00 UTC | ~345 дней |
| **1W**    | НЕТ ДАННЫХ | N/A |

**Вывод:** v3 API имеет **разную глубину для разных таймфреймов**. Это НЕ единое хранилище.
- **1m** — самая большая глубина (~424 дня, до марта 2025)
- **1d** — тоже большая (~345 дней)
- **5m, 15m, 1h** — всего 14-45 дней
- **1W** — пусто

### 2. v2 Swap Klines (`/openApi/swap/v2/quote/klines`)

| Таймфрейм | Самая ранняя свеча | Глубина (дней) |
|-----------|-------------------|----------------|
| **1m**    | 2024-12-16 01:14 UTC | ~501 день |

**Вывод:** v2 API даёт **ещё больше глубины**, чем v3 (~501 день против ~424 для 1m).  
Данные есть с декабря 2024 по настоящее время.

### 3. Spot Historical K-line (`/openApi/market/his/v1/kline`)

| Таймфрейм | Результат |
|-----------|-----------|
| **1m**    | Endpoint работает. Данные есть с 2025-03 |

**Вывод:** Spot Historical endpoint **работает**, но глубина примерно та же, что у v3 Swap.  
Формат ответа отличается — возвращает **массив**, а не объект с `data`.  
Потенциально может дать доступ к спотовым данным за больший период (не проверено).

### 4. Mark Price Klines (`/openApi/swap/v3/markPriceKlines`)

| Таймфрейм | Результат |
|-----------|-----------|
| **1m**    | Endpoint НЕ РАБОТАЕТ (другой формат или несуществующий endpoint) |

**Вывод:** Mark Price Klines endpoint не отвечает или требует других параметров.

## Неожиданное открытие

**v2 API даёт БОЛЬШЕ глубины, чем v3** для 1m таймфрейма:
- v3 1m: ~424 дня (с 2025-03-03)
- v2 1m: ~501 день (с 2024-12-16)

Это противоречит предположению, что v2 и v3 используют одно хранилище.

## Рекомендации

1. **Для backfill использовать v2 API** (`/openApi/swap/v2/quote/klines`) — он даёт больше данных
2. **Проверить v2 для других таймфреймов** (5m, 15m, 1h, 1d) — возможно, там тоже больше глубина
3. **Spot Historical K-line** — проверить с `startTime` за 2020-2023, возможно, спотовые данные хранятся дольше
4. **1W таймфрейм** — данные отсутствуют во всех endpoint'ах. Возможно, нужно агрегировать из 1d
